## 项目简介

**个人日程助手**是一款基于Qt框架开发的桌面应用程序，专为需要高效管理日程、查看农历信息和天气预报的用户设计。当前版本(v0.1.1)提供了完整的日程管理功能，包括日历展示、批量日程管理、农历转换和天气提醒。

**下一版本(v0.2预告)**将引入强大的多邮箱管理系统，基于SMTP/IMAP协议实现邮件发送、接收和日程智能关联功能，打造一体化的日程与邮件管理平台。

## 系统要求

### 最低要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10/11, macOS 10.14+, Linux (Ubuntu 18.04+) |
| 处理器 | 1 GHz 或更高 |
| 内存 | 2 GB RAM |
| 硬盘空间 | 200 MB 可用空间 |
| 显示器 | 1024x768 或更高分辨率 |

### 运行时依赖

- **Qt Runtime**: Qt 5.15+ 或 Qt 6.2+
- **Visual C++ Redistributable**: Visual Studio 2015-2022 (Windows)
- **OpenSSL**: 1.1.1+ (v0.2版本必需)

### 网络要求

- **互联网连接**: 天气API、邮件同步需要网络访问
- **防火墙设置**: 允许SMTP(25,465,587)、IMAP(143,993)、POP3(110,995)出站流量

## 安装部署

### Windows安装

#### 方法一：安装程序 (推荐)

1. 下载最新版本安装包 `PersonalDateAssisant-Setup-v0.1.1.exe`
2. 双击运行安装程序
3. 按照安装向导完成安装
4. 选择安装目录（建议使用默认目录）
5. 完成安装后，从开始菜单启动程序

#### 方法二：便携版

1. 下载便携版压缩包 `PersonalDateAssisant-Portable-v0.1.1.zip`
2. 解压到任意目录
3. 双击 `PersonalDateAssisant.exe` 启动程序
4. 数据存储在程序目录下的 `data` 文件夹

#### v0.2 OpenSSL依赖说明

> ⚠️ **v0.2版本特殊说明**
> 
> v0.2版本需要额外的OpenSSL运行时库支持邮件加密功能：
> 
> 1. 下载 [OpenSSL 1.1.1 for Windows](https://slproweb.com/products/Win32OpenSSL.html)
> 2. 安装运行时库
> 3. 或将 `libssl-1_1-x64.dll` 和 `libcrypto-1_1-x64.dll` 复制到程序目录

### 基础操作

#### 添加日程

1. 点击 **"+ 添加日程"** 按钮
2. 填写日程信息：
   - 标题（必填）
   - 描述（可选）
   - 开始时间
   - 结束时间（可选）
   - 优先级选择
   - 提醒时间设置
3. 点击 **"保存"** 完成

#### 查看日程

1. 点击日历上的任意日期
2. 显示该日期的所有日程
3. 双击日程可查看详情
4. 可直接在弹窗中编辑或删除

#### 编辑/删除日程

1. 双击日程项打开详情
2. 修改信息后点击 **"保存"**
3. 或点击 **"删除"** 移除日程
4. 批量删除：勾选多个日程后点击 **"批量删除"**

## 常见问题

### 安装问题

**Q: 安装失败，提示缺少DLL文件**
```
解决方案：
1. 安装 Visual C++ Redistributable
2. 下载并安装 Qt Runtime
3. 重新下载安装包
```

**Q: 杀毒软件阻止程序运行**
```
解决方案：
1. 将程序添加到杀毒软件白名单
2. 暂时关闭杀毒软件进行安装
3. 从官方网站下载避免被拦截
```

### 运行问题

**Q: 程序启动后无响应**
```
解决方案：
1. 检查显卡驱动是否最新
2. 尝试使用兼容模式运行
3. 查看日志文件排查错误
```

**Q: 天气信息显示"获取失败"**
```
解决方案：
1. 检查网络连接
2. 确认高德API Key有效
3. 查看API调用配额
```

**Q: 日历点击无反应**
```
解决方案：
1. 重新启动程序
2. 检查CalendarDelegate配置
3. 查看Qt Creator日志输出
```

### v0.2 邮件功能问题

**Q: SMTP连接超时**
```
解决方案：
1. 检查防火墙设置
2. 确认SMTP端口未被阻止
3. 尝试更换端口（587→465）
```

**Q: SSL证书验证失败**
```
解决方案：
1. 更新OpenSSL到最新版本
2. 确保证书链完整
3. 测试环境可忽略证书错误
```

**Q: Gmail发送失败**
```
解决方案：
1. Gmail需要开启"应用专用密码"
2. 设置→安全性→两步验证→应用密码
3. 使用生成的16位密码
```

**Q: IMAP同步很慢**
```
解决方案：
1. 限制同步邮件数量
2. 使用IDLE命令保持连接
3. 启用增量同步模式
```

### 数据问题

**Q: 如何备份数据？**
```
方法1：程序内备份
设置 → 数据管理 → 备份数据库

方法2：手动备份
找到数据文件：%APPDATA%/PersonalDateAssisant/schedules.db
复制备份到安全位置
```

**Q: 如何恢复数据？**
```
设置 → 数据管理 → 恢复数据库
选择备份文件进行恢复
```

- 📝 完善文档

### 开发指南

```markdown
# 开发环境设置
1. Fork 项目到你的GitHub账户
2. Clone 你fork的仓库
3. 创建功能分支：git checkout -b feature/your-feature
4. 进行开发并提交代码
5. 推送到你的仓库
6. 创建Pull Request

# 代码规范
- 遵循Qt编码规范
- 使用中文注释（可选）
- 确保代码通过lint检查
- 编写必要的单元测试
```

### 测试指南

```bash
# 运行单元测试
cd build
ctest --output-on-failure

# 运行特定测试
./tests/ScheduleTest

```
