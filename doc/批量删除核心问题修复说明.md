# 批量删除核心问题修复说明

## 修复日期
2026-04-07

## 问题描述

用户多次报告："批量日程删除功能依然没有实现"、"问题依旧"

## 问题现象

- 批量日程在列表中合并显示为 1 个条目
- 用户选中该条目并点击"批量删除"
- 系统显示将删除 N 个日程
- 但实际只删除了 1 个日程

---

## 根本原因分析

### 数据库中的数据结构

当创建 1 个 5 天批量日程时（04/01 - 04/05），数据库中会创建 **5 条独立的记录**：

| id | title | datetime | endDatetime | isBatch |
|----|----|----------|-------------|---------|
| 1 | 项目周 | **04-01** 09:00 | 04-05 18:00 | 1 |
| 2 | 项目周 | **04-02** 09:00 | 04-05 18:00 | 1 |
| 3 | 项目周 | **04-03** 09:00 | 04-05 18:00 | 1 |
| 4 | 项目周 | **04-04** 09:00 | 04-05 18:00 | 1 |
| 5 | 项目周 | **04-05** 09:00 | 04-05 18:00 | 1 |

**关键发现**：每条记录的 `datetime` 都不同，只有 `endDatetime`、`title`、`isBatch`、`priority` 相同！

---

## 问题根源

### 原 DELETE 查询（错误）

```cpp
DELETE FROM schedules 
WHERE isBatch = 1 
  AND title = ? 
  AND datetime = ?        // ❌ 问题所在！
  AND endDatetime = ? 
  AND priority = ?
```

**问题**：
- 当用户选中 ID=1 的记录（datetime = 04-01）
- DELETE 查询用 `datetime = 04-01` 去匹配
- **只能匹配到 1 条记录**（id=1）
- 其他 4 条记录的 datetime 是 04-02, 04-03, 04-04, 04-05，不会被删除！

---

## 修复方案

### 修正的 DELETE 查询

```cpp
DELETE FROM schedules 
WHERE isBatch = 1 
  AND title = ? 
  AND endDatetime = ?      // ✅ 只匹配相同的 endDatetime
  AND priority = ?
  // ✅ 不包含 datetime！
```

**原因**：
- 批量日程的所有记录都有 **相同的 endDatetime**
- 移除 `datetime` 条件后，查询会匹配所有 5 条记录
- ✅ 实现真正的批量删除

## 代码修改

### 修改文件
`DatabaseManager.cpp`

### 修改方法
1. `getDeletePreview()` - 删除预览
2. `deleteSchedulesWithDetails()` - 删除执行

### 关键修改 1：移除 batchKey 中的 datetime

```cpp
// 修改前 ❌
QString batchKey = QString("%1|%2|%3|%4").arg(title).arg(datetime).arg(endDatetime).arg(priority);

// 修改后 ✅
QString batchKey = QString("%1|%2|%3").arg(title).arg(endDatetime).arg(priority);
```

### 关键修改 2：修正 DELETE 查询

```cpp
// 修改前 ❌
deleteQuery.prepare("DELETE FROM schedules WHERE isBatch = 1 AND title = ? AND datetime = ? AND endDatetime = ? AND priority = ?");
deleteQuery.addBindValue(title);
deleteQuery.addBindValue(datetime);        // ❌ 这行删除
deleteQuery.addBindValue(endDatetime);
deleteQuery.addBindValue(priority);

// 修改后 ✅
deleteQuery.prepare("DELETE FROM schedules WHERE isBatch = 1 AND title = ? AND endDatetime = ? AND priority = ?");
deleteQuery.addBindValue(title);
// deleteQuery.addBindValue(datetime);      // ✅ 删除这行
deleteQuery.addBindValue(endDatetime);
deleteQuery.addBindValue(priority);
```

### 关键修改 3：查询实际天数

```cpp
// 修改前 ❌（通过日期计算）
int days = startDate.date().daysTo(endDate.date()) + 1;

// 修改后 ✅（查询数据库实际记录数）
QSqlQuery countQuery;
countQuery.prepare("SELECT COUNT(*) FROM schedules WHERE isBatch = 1 AND title = ? AND endDatetime = ? AND priority = ?");
countQuery.addBindValue(title);
countQuery.addBindValue(endDatetime);
countQuery.addBindValue(priority);

int actualDays = 0;
if (countQuery.exec() && countQuery.next()) {
    actualDays = countQuery.value(0).toInt();
}
```

## 数据一致性保证

### 1. 唯一标识符

批量日程使用以下字段组合作为唯一标识：
- `title`
- `endDatetime`
- `priority`
- `isBatch`

**不包括 `datetime`**，因为每条记录的 datetime 都不同。

### 2. 去重机制

使用 `QSet<QString>` 跟踪已处理的批次：
```cpp
QSet<QString> deletedBatches;

for (int id : ids) {
    QString batchKey = QString("%1|%2|%3").arg(title).arg(endDatetime).arg(priority);
    
    if (!deletedBatches.contains(batchKey)) {
        deletedBatches.insert(batchKey);
        // 执行删除
    }
}
```

**效果**：即使选中了同一批次的多个 ID，也只会删除一次。

### 3. 实际记录数统计

使用 `COUNT(*)` 查询实际记录数：
```cpp
SELECT COUNT(*) FROM schedules WHERE isBatch = 1 AND title = ? AND endDatetime = ? AND priority = ?
```

**效果**：显示准确的删除数量，而不是通过日期计算的理论天数。

## 总结

### 核心问题

**原问题**：DELETE 查询包含 `datetime` 字段，导致只删除 1 条记录

**根本原因**：批量日程的每条记录有相同的 `endDatetime` 但不同的 `datetime`

### 修复方案

1. ✅ 移除 batchKey 中的 `datetime`
2. ✅ 移除 DELETE 查询中的 `datetime` 条件
3. ✅ 使用 `COUNT(*)` 查询实际记录数
4. ✅ 保证预览和删除的一致性

### 修复效果

- ✅ 批量日程能够完整删除
- ✅ 显示准确的删除数量
- ✅ 用户看到和实际删除的一致
- ✅ 不存在部分删除的情况
