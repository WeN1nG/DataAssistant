# 日历格子染色功能

## 功能概述

为日历格子添加根据日程优先级染色的功能，让用户能够直观地看到每天的日程紧急程度。

## 优先级与颜色方案

| 优先级 | 名称 | 颜色 | RGB值 | 说明 |
|--------|------|------|-------|------|
| 2 | 紧急 | 淡红色 | rgb(255, 180, 180) | 最醒目，用于紧急日程 |
| 1 | 重要 | 淡黄色 | rgb(255, 255, 180) | 中等醒目，用于重要日程 |
| 0 | 一般 | 淡蓝色 | rgb(180, 200, 255) | 较淡，用于普通日程 |
| -1 | 无日程 | 浅灰色 | rgb(250, 250, 250) | 无日程的普通日期 |

## 核心特性

### 优先级覆盖规则

**重的颜色覆盖轻的颜色**

当某一天有多个不同优先级的日程时，只显示最高优先级的颜色：
- 如果有紧急日程 → 显示淡红色
- 如果有重要日程但无紧急 → 显示淡黄色
- 如果只有一般日程 → 显示淡蓝色
- 如果无日程 → 显示浅灰色

## 实现方案

### 1. CalendarDelegate.h 新增方法

```cpp
// 获取某日期的最高优先级
int getMaxPriority(const QDate& date) const;
```

### 2. CalendarDelegate.cpp 实现

#### 新增 getMaxPriority 方法

```cpp
int CalendarDelegate::getMaxPriority(const QDate& date) const
{
    int maxPriority = -1;  // -1 表示无日程
    for (const Schedule& schedule : m_schedules) {
        if (schedule.datetime.date() == date) {
            if (schedule.priority > maxPriority) {
                maxPriority = schedule.priority;
            }
        }
    }
    return maxPriority;
}
```

#### 修改 paint 方法

在 paint 方法中，根据 maxPriority 设置不同的背景色：

```cpp
void CalendarDelegate::paint(QPainter* painter, const QStyleOptionViewItem& option, const QModelIndex& index) const
{
    QDate date = dateFromIndex(index);
    bool isCurrentMonth = (date.month() == m_currentMonth);
    bool isToday = (date == m_today);
    int maxPriority = getMaxPriority(date);  // 获取最高优先级
    
    QColor backgroundColor;
    
    if (isToday) {
        // 今天：浅灰色 + 深色边框
        backgroundColor = QColor(245, 245, 245);
        // ... 其他设置
    } else if (!isCurrentMonth) {
        // 非当月：更浅的灰色
        backgroundColor = QColor(238, 238, 238);
        // ... 其他设置
    } else if (maxPriority == 2) {
        // 紧急：淡红色
        backgroundColor = QColor(255, 180, 180);
    } else if (maxPriority == 1) {
        // 重要：淡黄色
        backgroundColor = QColor(255, 255, 180);
    } else if (maxPriority == 0) {
        // 一般：淡蓝色
        backgroundColor = QColor(180, 200, 255);
    } else {
        // 无日程
        backgroundColor = QColor(250, 250, 250);
    }
    
    // 绘制背景
    painter->setBrush(backgroundColor);
    painter->drawRect(rect);
    // ... 其他绘制逻辑
}
```

## 示例场景

### 场景1：单一日程

- 某天只有1个紧急日程 → 显示淡红色 ✓

### 场景2：多个同优先级日程

- 某天有3个一般日程 → 显示淡蓝色 ✓
- 某天有2个重要日程 → 显示淡黄色 ✓

### 场景3：多个不同优先级日程

- 某天有1个紧急 + 2个重要 + 3个一般 → 显示淡红色 ✓（最高优先级）
- 某天有2个重要 + 3个一般 → 显示淡黄色 ✓（最高优先级）
- 某天有3个一般 + 无紧急/重要 → 显示淡蓝色 ✓

## 技术要点

### 1. 优先级比较

```cpp
// 优先级数值越大，越"重"
if (schedule.priority > maxPriority) {
    maxPriority = schedule.priority;
}
```

### 2. 颜色叠加规则

```
优先级 2 (紧急) > 优先级 1 (重要) > 优先级 0 (一般)
   ↓               ↓               ↓
淡红色         淡黄色         淡蓝色
```

### 3. 绘制顺序

paint 方法的绘制顺序确保：
1. 背景色先绘制
2. 今天边框后绘制（覆盖在背景上）
3. 文字最后绘制

## 用户体验优化

### 视觉效果

- **紧急日程最醒目**：淡红色背景吸引注意力
- **重要日程次之**：淡黄色提供中等提示
- **一般日程柔和**：淡蓝色不干扰但可识别
- **层次分明**：不同优先级一目了然

### 实际应用

- **快速识别**：一眼看出哪天有紧急事务
- **规划优先**：帮助用户优先处理重要日程
- **视觉平衡**：颜色柔和不刺眼，适合长时间使用
