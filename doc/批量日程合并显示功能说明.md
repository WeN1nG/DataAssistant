# 批量日程合并显示功能说明

## 功能实现日期
2026-04-07

## 功能需求
针对通过批量创建功能生成的日程项目，系统应确保在日程列表中**不显示多个独立条目**，而是**整合为单个日程条目**。该整合后的日程条目必须明确标明完整的起始日期信息，以清晰反映批量创建的日程范围。

---

## 实现方案

### 1. 批量日程识别机制

#### 识别逻辑
系统通过以下字段组合来识别同一批次的日程：
- `isBatch`：标识是否为批量日程
- `title`：日程标题
- `endDatetime`：结束日期时间
- `priority`：优先级（确保同一批次的日程优先级一致）

#### 避免重复显示
在 `ScheduleListDialog::loadSchedules()` 方法中实现：
```cpp
// 用于记录已显示的批量日程（避免重复显示）
QSet<int> displayedBatchIds;

// 遍历所有日程
for (const Schedule& schedule : allSchedules) {
    // 如果是批量日程且已经显示过（同一批次的其他日期），则跳过
    if (schedule.isBatch && schedule.id != -1) {
        bool alreadyDisplayed = false;
        for (int displayedId : displayedBatchIds) {
            Schedule displayedSchedule = dbManager->getScheduleById(displayedId);
            if (displayedSchedule.id != -1 && 
                displayedSchedule.title == schedule.title &&
                displayedSchedule.endDatetime == schedule.endDatetime &&
                displayedSchedule.priority == schedule.priority) {
                alreadyDisplayed = true;
                break;
            }
        }
        
        if (alreadyDisplayed) {
            continue; // 跳过已显示的批量日程条目
        }
        
        // 标记这个批次为已显示
        displayedBatchIds.insert(schedule.id);
    }
    
    // 创建日程条目...
}
```

## 技术实现细节

### 1. 数据结构

```cpp
struct Schedule {
    int id;                      // 日程ID
    QString title;                // 标题
    QString description;         // 描述
    QDateTime datetime;          // 开始日期时间
    QDateTime endDatetime;       // 结束日期时间（批量日程）
    bool isBatch;                 // 是否为批量日程
    int priority;                // 优先级 (0=一般, 1=重要, 2=紧急)
    int reminderMinutes;          // 提前提醒分钟数
    bool completed;              // 是否完成
    QColor color;                 // 颜色
};
```

### 2. 关键算法

#### 批量日程合并算法
```
输入：allSchedules（所有日程列表）
输出：filteredSchedules（合并后的日程列表）

1. 清空 displayedBatchIds
2. 对 allSchedules 按日期升序排序
3. 对于每个 schedule：
   a. 如果不是批量日程，直接添加
   b. 如果是批量日程：
      i. 检查是否已在 displayedBatchIds 中
      ii. 如果不在，添加并记录到 displayedBatchIds
      iii. 如果在，跳过
4. 返回合并后的列表
```

### 3. 性能优化

- 使用 `QSet<int>` 存储已显示的批次ID，O(1) 查找时间
- 在创建日程条目之前进行合并判断，避免重复创建UI元素
- 保持原有的排序逻辑，确保日程按时间顺序显示
