# 农历日期计算修复报告

## 修复时间
2026-04-05

## 问题概述

系统中农历日期计算存在严重不准确的问题，原有实现使用简化算法，导致：
- 农历节日识别错误
- 闰月处理完全缺失
- 日期转换存在1-2天误差
- 节气计算不准确

## 问题分析

### 原实现问题

原实现在Widget.cpp中的`calculateLunarDate`函数存在以下问题：

1. **基准日期错误**
   - 使用2000年1月6日作为基准，但实际不是腊月初一
   - 基准日期：2000年1月6日（错误地设为腊月初一）

2. **算法过于简化**
   ```cpp
   // 所有月份都按30天计算
   int monthDays = (lunarMonth == 12) ? 30 : 30;
   ```
   - 农历月份实际有29天（小月）或30天（大月）
   - 没有考虑闰月

3. **节日检测逻辑错误**
   ```cpp
   // 使用公历日期检测农历节日
   if (month == 1 && day == 1) return "春节";  // 错误！
   ```
   - 应该使用农历日期判断，而不是公历日期

4. **闰月处理缺失**
   - 完全不支持闰月
   - 无法正确处理闰年农历日期

5. **年份计算错误**
   - 没有正确计算农历年份的推进
   - 没有使用准确的农历数据

## 修复方案

### 1. 创建LunarCalendar类

创建了专门的`LunarCalendar`类（[LunarCalendar.h](file:///c:/CODE/QTS/Projects/DateAssistant/DataAssistant/LunarCalendar.h)和[LunarCalendar.cpp](file:///c:/CODE/QTS/Projects/DateAssistant/DataAssistant/LunarCalendar.cpp)），包含：

#### 数据结构
```cpp
struct LunarDate {
    int year;       // 农历年
    int month;      // 农历月（1-12，闰月为负数表示）
    int day;        // 农历日（1-30）
    bool isLeap;    // 是否闰月
};
```

#### 核心功能
- `gregorianToLunar()` - 公历转农历
- `lunarToGregorian()` - 农历转公历
- `getLunarDateString()` - 获取农历日期字符串
- `getSolarTerm()` - 获取节气信息
- `getHoliday()` - 获取农历节日

### 2. 农历数据表

使用预计算的农历数据表（1900-2100年），每条数据包含：
- **闰月信息**：高4位存储闰月月份（0表示无闰月）
- **月份大小**：每12位表示12个月的大小（0=小月29天，1=大月30天）

```cpp
m_lunarInfo[117] = 0x6ad5;  // 2017年，闰六月
// 高4位：0x6 = 6月闰
// bit 0-11：110101010110 = 12个月的天数
```

### 3. 修正集成到Widget

更新Widget.h和Widget.cpp，使用新的LunarCalendar类：

```cpp
// Widget.h
private:
    LunarCalendar *m_lunarCalendar;

// Widget.cpp
void Widget::updateDateInfo() {
    QDate today = QDate::currentDate();
    
    QString lunarDate = m_lunarCalendar->getLunarDateString(today);
    ui->lunarLabel->setText(lunarDate);
    
    QString solarTerm = m_lunarCalendar->getSolarTerm(today);
    if (!solarTerm.isEmpty()) {
        ui->solarTermLabel->setText(solarTerm);
    }
}
```

## 技术说明

### 农历数据编码格式

LunarInfo使用16位整数编码一年的农历信息：

```
┌─────────────────────────────────────┐
│  Bit 15-12  │  Bit 11-0            │
├─────────────┼──────────────────────┤
│  闰月月份    │  12个月的天数         │
│  (0=无闰月) │  (0=29天, 1=30天)    │
└─────────────┴──────────────────────┘
```

例如：2017年 = 0x6ad5
- 0x6 = 闰六月
- 0xad5 = 12个月的天数分布

### 核心算法

#### 公历转农历
1. 计算从基准日期（1900年1月31日）到目标日期的天数
2. 逐年减去农历年的天数，找到对应的农历年
3. 逐年减去农历月的天数，找到对应的农历月
4. 处理闰月特殊情况

#### 农历转公历
1. 累加从基准年到目标年前的所有天数
2. 累加目标年中从正月到目标月的所有天数
3. 加上闰月天数（如适用）
4. 加上目标日期

2. ✅ **节日识别**：正确识别主要农历节日（春节、元宵、端午、中秋等）

4. ⚠️ **节气计算**：基础功能正常，但精度有待提高

- 农历日期的显示和转换

- 日期的双向精确转换
